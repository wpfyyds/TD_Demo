# 塔防肉鸽游戏技术方案文档

## 1. 项目概述

### 1.1 项目名称
像素塔防肉鸽 (Pixel Tower Defense Roguelike)

### 1.2 项目类型
塔防 + 肉鸽 (Roguelike) 混合类型游戏

### 1.3 技术栈
- **游戏引擎**: Unity 2022 LTS
- **游戏框架**: Game Framework (https://github.com/EllanJiang/GameFramework)
- **热更新技术**: xLua (https://github.com/Tencent/xLua.git)
- **渲染技术**: Unity Shader Graph + 自定义Shader
- **架构模式**: ECS (Entity Component System)
- **艺术风格**: 像素风格 (Pixel Art)

### 1.4 核心特性
- 程序化生成的像素风格场景和敌人
- 基于Game Framework的模块化架构
- 使用xLua实现防御塔和敌人的热更新
- 基于ECS的高性能动画和特效处理
- 肉鸽元素：随机关卡、永久升级、死亡重置

## 2. Game Framework框架搭建方案

### 2.1 框架集成步骤
1. **下载Game Framework**:
   - 从GitHub下载最新版本: https://github.com/EllanJiang/GameFramework
   - 解压后将Assets目录内容复制到Unity项目中

2. **基础配置**:
   - 创建Game Framework启动场景
   - 配置全局配置文件
   - 设置资源加载路径

3. **核心模块配置**:
   - **实体管理**: 配置实体工厂和对象池
   - **流程管理**: 创建游戏流程状态机
   - **资源管理**: 配置异步资源加载策略
   - **事件系统**: 注册游戏事件

### 2.2 游戏流程设计
```
Bootstrap流程 -> 资源检查流程 -> 主菜单流程 -> 游戏流程 -> 结算流程
```

### 2.3 模块划分
- **Core**: 核心游戏逻辑
- **UI**: 用户界面
- **Entity**: 游戏实体（塔、敌人、子弹）
- **AI**: 敌人AI和塔AI
- **Effect**: 特效系统
- **Audio**: 音频系统
- **Config**: 配置管理

## 3. Unity Shader特效实现方案

### 3.1 敌人死亡特效
**Shader类型**: 粒子Shader
**效果描述**:
- 像素风格的爆炸效果
- 颜色渐变和透明度变化
- 粒子大小和速度随机化

**关键特性**:
- 使用噪声纹理实现像素化效果
- 支持颜色渐变和闪烁效果
- 性能优化：减少过度绘制

### 3.2 塔防攻击特效
**Shader类型**: 线条/光束Shader
**效果描述**:
- 像素风格的攻击光束
- 颜色根据塔类型变化
- 速度和轨迹动画

**关键特性**:
- 使用UV动画实现光束效果
- 支持多种攻击类型（激光、炮弹、魔法）
- 与ECS系统集成

### 3.3 子弹特效
**Shader类型**: 精灵Shader
**效果描述**:
- 像素风格的子弹飞行效果
- 拖尾和发光效果
- 命中目标时的爆炸效果

**关键特性**:
- 支持多种子弹类型
- 与物理系统集成
- 性能优化：对象池管理

## 4. xLua热更新技术方案

### 4.1 热更新架构

   - 创建Lua脚本目录结构
   - 配置xLua环境
   - 实现Lua脚本加载器

   - 定义C#接口供Lua调用
   - 注册Lua回调函数
   - 实现数据转换

### 4.2 热更新内容
- **防御塔逻辑**:
  - 攻击逻辑
  - 升级系统
  - 特殊能力

- **敌人AI**:
  - 移动路径
  - 攻击行为
  - 特殊技能

### 4.3 热更新流程
1. 游戏启动时加载基础Lua脚本
2. 运行时检测更新包
3. 下载并替换Lua脚本
4. 重新加载受影响的游戏对象

## 6. ECS框架动画和特效处理方案

### 6.1 ECS架构设计
**核心组件**:
- TransformComponent: 位置、旋转、缩放
- RenderComponent: 渲染信息
- AnimationComponent: 动画状态
- EffectComponent: 特效参数

**系统设计**:

- 支持混合动画

### 6.3 特效处理
- 粒子系统集成
- 特效池管理
- 性能优化策略
